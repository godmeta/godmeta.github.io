DMeta: Decentralized Metaverse

DMetaHash: Base64 53-bit Hash




Decentralized Metaverse could be something very abstract to many people.

However, it can be constructed based on something very practical and specific — DMetaHash.

DMetaHash is a 53-bit integer hash encoded in Base64 string.


Users may enter some random text in the text box on the left above and press the DMetaHash button in the middle, OR enter the following in browser console to verify the DMetaHash function for themselves:

  • bnToB64(cyrb53($('#input2').val())))
  • bnToB64(cyrb53('anystring'))

We may use the following as input to DMetaHash function:

  • Cryptocurrency Wallet Address, e.g. BTC, TRX, TRX-USDT
  • RSA Public Key
  • RSA Private Key
  • Google Document address, or any web address
  • A Contract between Two or More Parties
  • A complete web page source code
  • A complete or partial program source code
  • One or more DMetaHashes

Please click on the following address or DMetaHash button (BfMGjHy9zQ==) for detailed explanations of each of the use cases above:

As the world is hyping about Metaverse, few have raised concern, let alone proposed solutions to the real menace — CENTRALIZATION.

The first innovation of DMeta comes from an unsuspected category: hash functions.

By generating hashes of URI, we create alternative, decentralized indices of objects or pages, previously controlled by centralized platforms, such as Google Drive, Twitter, etc.

We call these hashes of URI objects HURRAY, as objects are equivalent to arrays in most programming languages.

The first practical application HURRAY is HYPERCOMPOSITION. Although sounding avant garde, hypercomposition is simply the process of composing hypertext, which has been hindered by the many layers of tools, instead of helping the writer organize his (her, etc.) thoughts and write freely.

For example, when author A (Adam) first composed an article (A1) on Google Document, it was saved with the following URI:
https://docs.google.com/document/d/1U_fN6fbhdHBUnbibeWhgwtDbDFLSeSk3wu60VVo9ykw/

A few days later, Adam wrote another article (A2), saved with the folloing URI: https://docs.google.com/document/d/1ZfeoglyOo5FLjplUDQ2kgR-Umb0qbql_W7KqpykRaP0/

When Adam wishes to copy parts of articles A1 and A2 to party B (Brian), he may use HURRAY of A1 and A2 as reference to share with Brian.
